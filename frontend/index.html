Aggregate Functions

COUNT, SUM, AVG, MIN, MAX

Aggregate functions perform calculations across multiple rows to return a single summary value. Combined with GROUP BY, they enable powerful data analysis and reporting.

Subqueries (nested queries) are SELECT statements embedded inside another SQL statement. They can appear in WHERE, HAVING, FROM, or SELECT clauses, enabling complex filtering and comparison logic.

JOIN operations combine rows from two or more tables based on related columns. Different join types (INNER, LEFT, RIGHT, CROSS, SELF) control which rows appear in the result when matches are missing.

Set operations combine the results of two or more SELECT statements. UNION merges and deduplicates, UNION ALL keeps duplicates, INTERSECT finds common rows, and EXCEPT finds rows in one set but not another.

Views are virtual tables defined by SQL queries. They simplify complex queries, provide data abstraction, and can enforce security by exposing only certain columns. Views don't store data — they compute results on each query.

Stored functions are reusable SQL routines that accept parameters, perform logic, and RETURN a single value. They can be used inside SELECT statements just like built-in functions.

Stored procedures are named SQL programs stored in the database. Unlike functions, they can modify data, return multiple result sets, and use IN/OUT/INOUT parameters. They're invoked with CALL.

Triggers are SQL code that automatically executes in response to INSERT, UPDATE, or DELETE events on a table. They run BEFORE or AFTER the event and are used for validation, auditing, and enforcing business rules.

Cursors allow row-by-row processing of query results inside stored procedures. They're used when set-based operations aren't sufficient and you need iterative logic for each row.

Database normalization organizes tables to minimize redundancy and prevent anomalies. The process involves decomposing tables through normal forms (1NF → 2NF → 3NF), each eliminating specific types of dependencies.

Concurrency control manages simultaneous database access. Transactions ensure ACID properties, isolation levels control what concurrent sessions can see, and locks prevent conflicting modifications.
